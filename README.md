# 🛳️ SAR衛星データによる船舶検出の学習ポートフォリオ

Sentinel-1 SARデータを使って港湾の船舶を検出する手法を、Google Earth Engine上で実装・検証した学習記録です。
東京湾（横浜港・川崎港沖）を題材に、CA-CFARアルゴリズムによる船舶自動検出を試みました。

> **このリポジトリの位置づけ**  
> 衛星データ解析を学習中の段階で作成したものです。技術的な完成度よりも、SARという技術を理解し、実データで手を動かしてみた過程を記録することを目的にしています。  
> コードの実装にはAIツールを活用しており、処理の意図と仕組みは自分の言葉で説明できる状態を目指しました。

---

## 📌 このリポジトリで取り組んだこと

海運大国である日本では、港湾の船舶動態把握が物流・安全管理の基本になります。船舶位置情報はAIS（自動船舶識別装置）で管理されていますが、AISは小型船には搭載義務がなく、意図的にオフにすることもできます。

SARは金属製の船体を電波で捉えるため、AISのオンオフに関係なく船舶を検出できる——これを自分の手で試してみました。

具体的には、海面の後方散乱統計を推定し、それより有意に明るい輝点を船舶候補として抽出するCA-CFAR法を、Google Earth Engine上で実装しています。

## 📊 SARで船舶が検出できる仕組み（学習メモ）

```
海面の特性：
  穏やかな海面は電波を鏡面反射するため、SAR画像上で暗く映る

船舶の特性：
  金属構造が電波を強く反射するため、海面背景の中で明るい輝点として映る

検出の仕組み（CFAR法）：
  海面の明るさのばらつきを統計的に推定し、
  それより有意に明るいまとまりを船舶候補として抽出する

海面マスクの適用：
  陸地の誤検出を除去するため、SAR輝度が低いエリア（海面）のみに
  検出範囲を絞り込む（閾値：-17dB）
```

実装の流れ：

1. 対象期間のSentinel-1 GRDシーンをGEEから取得し、メディアン合成（27シーン）
2. dB変換して後方散乱係数を計算
3. 海面マスクを適用し、陸地を除外
4. CA-CFAR法で局所的に明るい輝点を抽出
5. AIS照合シミュレーションで検出結果を「照合済み／未照合」に分類

CFAR法のパラメータ（ガード窓・参照窓のサイズ、誤警報確率）や閾値の設定は、まだ改善の余地があり、実運用レベルにするには専門家の検証が必要だと認識しています。

## 🛰️ 無料公開SAR と 商用高分解能SAR の違い（理解メモ）

本解析で使用したSentinel-1（無料公開）と、商用の高分解能SARでは、できることに差があります。

| 比較軸 | Sentinel-1（本解析で使用） | 商用高分解能SAR |
|--------|---------------------------|----------------|
| 分解能 | 5〜20m | 46cm〜1m級 |
| 観測頻度（同一地点） | 数日〜1週間 | コンステレーション化により高頻度化が進行中 |
| 小型船の識別 | 難しい | 詳細な把握が期待される |
| 船種識別 | 困難 | 船体の輪郭・向きまで識別できる可能性 |
| 利用コスト | 無料公開 | 商用ライセンス |
| 夜間・悪天候 | 可 | 可 |

Sentinel-1でも大型船は検出できますが、小型漁船や工作船の識別には分解能が足りません。本解析で構築した処理パイプラインは、より高分解能なSARデータが入手できるようになった際にも、同じ考え方で応用できるはずだと考えて作っています。
